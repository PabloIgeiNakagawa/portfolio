Contributing to mi-portfolio
Versión en español

Guía de contribución
- Rama y flujo de trabajo
  - Crea una rama por tarea: por ejemplo feat/lazy-loading o fix/hero-img
  - Haz commits pequeños y enfocados
- Mensajes de commit
  - En español; formato recomendado: <Tipo>: Descripción breve
  - Tipos sugeridos: feat, fix, perf, docs, chore, refactor
- Cuerpo del commit
  - El cuerpo debe explicar qué cambió y por qué
  - Incluye cómo verificar o probar los cambios
- Pruebas
  - Ejecuta: npm install, npm run build, y npm run dev
- Revisión y calidad
  - Pide revisión si trabajas en equipo
- Seguridad
  - No incluir secretos ni credenciales
- Estilo
  - Mantén consistencia en el formato y usa puntuación
- Ejemplos
  - feat: Añade lazy loading a imágenes
  - fix: Corrige ruta de imagen en Hero
  - perf: Optimiza tamaño de assets en Home
